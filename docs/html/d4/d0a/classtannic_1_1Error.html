Detailed Description

Represents serious, typically unrecoverable problems.

The Error class is used for critical failures in the program or environment that should not normally be caught and handled. Examples include out-of-memory conditions or hardware failures.

Note
While technically catchable, errors are conceptually distinct from exceptions and should rarely be handled in normal application logic.
